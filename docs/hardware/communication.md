# 通信接口

## CAN

STM32 CAN 接口使用 `PA11/CAN_RX` 和 `PA12/CAN_TX`，经 TJA1051 收发器转换到 `CANH/CANL`。板卡未集成 120 Ω 终端电阻，必须根据其在总线中的位置外接终端。

::: danger CAN 接线
断电接线并确认 CANH/CANL 不反接。双终端网络断电测量应约为 60 Ω；若接近 120 Ω，通常只安装了一个终端；若明显更小，可能终端过多或存在短路。
:::

## SBUS

SBUS 接收使用 `PA3 / USART2_RX`，信号为反相 TTL。典型串口参数为：

```text
波特率：100000
数据位：8
校验：Even（偶校验）
停止位：2
```

SBUS 相关电路依赖 5 V，使用时保持 USB 供电。软件需要按协议帧头、长度和通道位域解包，并设置失联超时保护。

## USART

常用 USART1 引脚为 `PA9/TX` 与 `PA10/RX`，通过 GH1.25-4P 接口引出。连接 USB 转串口或其他控制器时，应交叉连接 TX/RX 并共地，同时确认对端是 3.3 V TTL 电平。

## 总线排查顺序

1. 电源、电平、共地和线序。
2. 引脚复用、时钟和 GPIO 模式。
3. 波特率、帧格式或 CAN 位时序。
4. 用逻辑分析仪/示波器确认线上是否真的有波形。
5. 最后再检查协议解析代码。
