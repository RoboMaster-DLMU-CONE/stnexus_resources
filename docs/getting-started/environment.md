# 开发环境

## 建议工具链

| 工具 | 用途 | 是否必需 |
| --- | --- | --- |
| STM32CubeMX | 配置时钟、引脚和外设并生成初始化代码 | 建议 |
| STM32CubeIDE | 编辑、编译、下载与基础调试 | 二选一 |
| CMake + Arm GNU Toolchain | 使用仓库中已有的 CMake 工程 | 二选一 |
| J-Link Software | 识别板载 Link，配合 Ozone/J-Flash 调试 | 建议 |
| Git | 获取例程并管理自己的修改 | 必需 |

## 环境检查

1. 使用**支持数据传输**的 Type-C 线连接开发板。
2. 在设备管理器中确认调试设备被系统识别。
3. 打开示例工程，目标芯片选择 `STM32F103C8T6`。
4. 确认系统时钟采用 8 MHz HSE，经 PLL 倍频至 72 MHz。
5. 首次构建保持原工程配置，不要同时升级 HAL 包或修改链接脚本。

::: info 关于例程
源资料中已经提供 GPIO LED、OLED UI 和 FreeRTOS 综合工程；其余目录目前主要是训练主题骨架。本站为这些主题补充了实验目标和实现路线，具体工程代码会随仓库继续完善。
:::

## 常见识别问题

- **只有供电、没有调试设备**：更换数据线或 USB 端口。
- **无法连接目标芯片**：检查 BOOT 跳线、复位状态和 SWD 占用情况。
- **下载后程序不运行**：确认启动模式恢复到用户 Flash，并按一次复位键。
- **时序全部异常**：核对 HSE 与 PLL 配置，确认系统确实运行在 72 MHz。
